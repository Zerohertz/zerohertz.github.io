Introduction

Apache Airflow

  • Definition
    • мҳӨн”Ҳ мҶҢмҠӨлЎң к°ңл°ңлҗң лҚ°мқҙн„° нҢҢмқҙн”„лқјмқё кҙҖлҰ¬ лҸ„кө¬
    • Workflow мһҗлҸҷнҷ”мҷҖ лҚ°мқҙн„° мІҳлҰ¬лҘј мң„н•ң н”Ңлһ«нҸј
  • Features
    • мң м—°н•ң нҢҢмқҙмҚ¬ н”„л Ҳмһ„мӣҢнҒ¬лҘј нҶөн•ң мү¬мҡҙ лҚ°мқҙн„° нҢҢмқҙн”„лқјмқё кө¬м¶•
    • лӢӨм–‘н•ң л№Ңл”© лё”лЎқмқ„ нҶөн•ң мөңмӢ  кё°мҲ  м—°кІ°
    • ліөмһЎн•ң лҚ°мқҙн„° мІҳлҰ¬ мһ‘м—… scheduling л°Ҹ monitoring
    • Data engineering, ETL (Extract, Transform, Load), data migration, data analysis л“ұ лӢӨм–‘н•ң мһ‘м—…м—җм„ң нҷңмҡ©
  • Components

      • Gunicorn кё°л°ҳмқҳ Flask м„ңлІ„

      • мһ‘м—…мқҳ мӢӨн–ү мғҒнғң, log, scheduling л“ұ нҷ•мқё к°ҖлҠҘ
    • Scheduler
      • Multi threadлЎң кө¬м„ұлҗң python process
      • WorkflowлҘј мӢӨн–үн•ҳкё° мң„н•ҙ м •мқҳлҗң мӢңк°„лҢҖмҷҖ мЈјкё°м—җ л”°лқј мһ‘м—…л“Өмқ„ scheduling
      • м •мқҳлҗң мҲңм„ңм—җ л”°лқј мӢӨн–үлҗҳлҠ” schedule мғқм„ұ
    • Database
      • лӘЁл“  DAG л°Ҹ taskмқҳ metadata м ҖмһҘ
      • лӢӨм–‘н•ң database м§Җмӣҗ (SQLite, MySQL, PostgreSQL)

    • Worker
      • Workflowмқҳ taskлҘј мӢӨм ңлЎң мӢӨн–үн•ҳлҠ” computing source
      • к°Ғ workerлҠ” лҸ…лҰҪм ҒмңјлЎң task мҲҳн–ү
      • н•„мҡ”м—җ л”°лқј workload л¶„мӮ° к°ҖлҠҘ
    • Triggerer
      • Deferrable operatorлҘј м§Җмӣҗн•ҳлҠ” лі„лҸ„мқҳ process
      • мӮ¬мҡ© мӢң лі„лҸ„ мӢӨн–ү (м„ нғқм Ғ мӮ¬мҡ©)
  • Etc.
    • DAG (Directed Acyclic Graph)
      • WorkflowлҘј м •мқҳн•ҳлҠ” code нҢҢмқј
      • мһ‘м—… л°Ҹ мқҳмЎҙм„ұ м •мқҳ
    • Task
      • OperatorsлҘј нҶөн•ҙ мғқм„ұлҗң мһ‘м—… лӢЁмң„ (DAGм—җм„ң н•ҳлӮҳмқҳ л…ёл“ң)
      • лҸ…лҰҪм ҒмңјлЎң мӢӨн–үлҗҳлҠ” н•ҳлӮҳмқҳ лӢЁмң„ мһ‘м—…
    • Operator
      • к°ңлі„ мһ‘м—…мқ„ лӮҳнғҖлӮҙлҠ” class (BashOperator, PythonOperator)
      • нҠ№м • мһ‘м—… мҲҳн–ү мӢң мӮ¬мҡ©
    • Sensor
      • мҷёл¶Җ мЎ°кұҙ нҳ№мқҖ мқҙлІӨнҠё л°ңмғқк№Ңм§Җ лҢҖкё° нӣ„ нҠ№м • мЎ°кұҙ л§ҢмЎұ мӢң task мӢӨн–ү
      • мҷёл¶Җ мӢңмҠӨн…ңмқҳ лҚ°мқҙн„° мң л¬ҙ нҷ•мқё л°Ҹ нҢҢмқј мғқм„ұ м—¬л¶Җ нҷ•мқё л“ұ мһ‘м—… мҲҳн–ү

Data Pipeline

Data pipelineмқ„ нҶөн•ҙ мӣҗн•ҳлҠ” кІ°кіјлҘј м–»кё° мң„н•ҙм„ңлҠ” API нҳём¶ң, лҚ°мқҙн„° м „мІҳлҰ¬ л“ұ лӢӨм–‘н•ң taskк°Җ мҲҳн–үлҗңлӢӨ.
лҳҗн•ң к°Ғ taskл“ӨмқҖ мқҳмЎҙм„ұмқҙ мЎҙмһ¬н•ҳм—¬ м •н•ҙм§„ мҲңм„ңлҢҖлЎң м§„н–үлҗҳм–ҙм•ј н•ңлӢӨ.
мқҙлҘј к°„лӢЁн•ҳкІҢ мқҙн•ҙн•ҳкё° мң„н•ҙ task к°„ мқҳмЎҙм„ұмқ„ м•„лһҳ мҳҲмӢңмҷҖ к°ҷмқҙ directed graph (л°©н–Ҙм„ұ к·ёлһҳн”„)лЎң лӮҳнғҖлӮёлӢӨ.

API нҳём¶ң $\rightarrow$ лҚ°мқҙн„° м „мІҳлҰ¬ $\rightarrow$ лӘЁлҚё мҳҲмёЎ

мң„мҷҖ к°ҷмқҖ нҳ•нғңлҘј Directed Acycllic Graph, DAG (л°©н–Ҙм„ұ л№„мҲңнҷҳ к·ёлһҳн”„)лқј м№ӯн•ңлӢӨ.
Taskмқҳ мқҳмЎҙм„ұмқ„ лӮҳнғҖлӮҙлҠ” $\rightarrow$мқҳ directed edge (лҒқм җ)мқҳ л°ҳліө нҳ№мқҖ мҲңнҷҳмқ„ н—Ҳмҡ©н•ҳм§Җ м•ҠлҠ”лӢӨ. (мҲңнҷҳ к·ёлһҳн”„лҠ” deadlock л°ңмғқ)

м Ҳм°Ём Ғ script pipelineм—җ л№„н•ҙ graph pipelineмқҖ мң„мҷҖ к°ҷмқҖ кІҪмҡ° taskлҘј лі‘л ¬лЎң мӢӨн–үн•  мҲҳ мһҲкё° л•Ңл¬ём—җ лҚ” нҡЁмңЁм ҒмқҙлӢӨ.
лҳҗн•ң м „мІҙ мһ‘м—…мқ„ н•ҳлӮҳмқҳ monolithic script лҳҗлҠ” processлЎң кө¬м„ұн•ҳлҠ” кІғмқҙ м•„лӢҲлқј taskлЎң лӘ…нҷ•нһҲ л¶„лҰ¬н•ҳкё° л•Ңл¬ём—җ мӨ‘к°„мқҳ taskк°Җ мӢӨнҢЁн•  л•Ң м „мІҙ scriptлҘј мһ¬мӢӨн–үн•ҳлҠ” кІғмқҙ м•„лӢҢ мӢӨнҢЁн•ң taskл§Ң мһ¬мӢӨн–үн•ңлӢӨ.

ETL vs. ELT

ETL (Extract, Transform, Load) ELT (Extract, Load, Transform)
м •мқҳ лҚ°мқҙн„° м¶”м¶ң (Extract) $\rightarrow$ ліҖнҷҳ (Transform) $\rightarrow$ лҚ°мқҙн„° м Ғмһ¬ (Load) лҚ°мқҙн„° м¶”м¶ң (Extract) $\rightarrow$ лҚ°мқҙн„° м Ғмһ¬ (Load) $\rightarrow$ ліҖнҷҳ (Transform)
ліҖнҷҳ мІҳлҰ¬ мң„м№ҳ м¶”м¶ң нӣ„ ліҖнҷҳ мһ‘м—…мқ„ мҲҳн–ү м Ғмһ¬ нӣ„ ліҖнҷҳ мһ‘м—…мқ„ мҲҳн–ү
лҚ°мқҙн„° м ҖмһҘ мң„м№ҳ м Ғмһ¬ м „м—җ ліҖнҷҳлҗң лҚ°мқҙн„°лҘј мӨ‘к°„ м ҖмһҘмҶҢм—җ м ҖмһҘ м Ғмһ¬ нӣ„м—җ мӣҗмӢң лҚ°мқҙн„°лҘј лҚ°мқҙн„° мӣЁм–ҙн•ҳмҡ°мҠӨм—җ м ҖмһҘ
лҚ°мқҙн„° мҡ©лҹү ліҖнҷҳлҗң лҚ°мқҙн„° мҡ©лҹүмқҙ нҒҙ мҲҳ мһҲмқҢ мӣҗмӢң лҚ°мқҙн„°лҘј м ҖмһҘн•ҳкё° л•Ңл¬ём—җ мҡ©лҹүмқҙ нҒј
ліҖнҷҳ мӢңм җ м¶”м¶ң нӣ„ ліҖнҷҳ мһ‘м—…мқ„ нҶөн•ҙ н•„мҡ”н•ң лҚ°мқҙн„°л§Ң м Ғмһ¬ м Ғмһ¬ нӣ„ ліҖнҷҳ мһ‘м—…мқ„ нҶөн•ҙ н•„мҡ”н•ң лҚ°мқҙн„°лҘј м¶”м¶ң
мһҘм җ рҹ‘Қ м Ғмһ¬ мӢңк°„мқҙ м§§мқҢ
рҹ‘Қ мӨ‘к°„ м ҖмһҘмҶҢлҘј нҶөн•ң мһ¬мӢңлҸ„ л°Ҹ мҳӨлҘҳ мІҳлҰ¬ к°ҖлҠҘ
рҹ‘Қ м Ғмһ¬ мӢң лҚ°мқҙн„° ліҖкІҪ м—ҶмқҢ (лҚ°мқҙн„° н’Ҳм§Ҳ мң м§Җ)		 рҹ‘Қ ліҖнҷҳ мӢңк°„мқҙ м§§мқҢ
рҹ‘Қ л№ лҘё ліҖнҷҳ мһ‘м—… к°ҖлҠҘ
рҹ‘Қ ліҖнҷҳлҗң лҚ°мқҙн„°лҘј м§Ғм ‘ нҷңмҡ© к°ҖлҠҘ
лӢЁм җ рҹ‘Һ мӨ‘к°„ м ҖмһҘмҶҢ кҙҖлҰ¬ н•„мҡ”
рҹ‘Һ м Ғмһ¬ мӢңк°„мқҙ мғҒлҢҖм ҒмңјлЎң кёё мҲҳ мһҲмқҢ		 рҹ‘Һ м Ғмһ¬ мӢңк°„мқҙ мғҒлҢҖм ҒмңјлЎң кёё мҲҳ мһҲмқҢ
рҹ‘Һ мӨ‘к°„ м ҖмһҘмҶҢк°Җ м—ҶмңјлҜҖлЎң мҳӨлҘҳ мІҳлҰ¬м—җ м ңм•Ҫмқҙ мһҲмқҢ

Data Lake vs. Data Warehouse

Data Lake Data Warehouse
м •мқҳ л№„кө¬мЎ°м Ғмқё мӣҗмӢң лҚ°мқҙн„° м ҖмһҘмҶҢ кө¬мЎ°нҷ”лҗң лҚ°мқҙн„° м ҖмһҘмҶҢ
лҚ°мқҙн„° мң нҳ• мӣҗмӢң лҚ°мқҙн„° (л№„м •нҳ•, л°ҳм •нҳ•, м •нҳ•) м •нҳ• лҚ°мқҙн„° (кө¬мЎ°нҷ”лҗң н…Ңмқҙлё” нҳ•нғң)
лҚ°мқҙн„° м ҖмһҘ нҳ•мӢқ нҢҢмқј мӢңмҠӨн…ң (HDFS, AWS S3 л“ұ) кҙҖкі„нҳ• лҚ°мқҙн„°лІ мқҙмҠӨ (RDBMS)
лҚ°мқҙн„° мІҳлҰ¬ м ‘к·јм„ұ мҠӨнӮӨл§ҲлӮҳ мҠӨнӮӨл§Ҳ ліҖнҷҳ м—Ҷмқҙ м ‘к·ј к°ҖлҠҘ мӮ¬м „ м •мқҳлҗң мҠӨнӮӨл§Ҳм—җ л”°лқј м ‘к·ј
лҚ°мқҙн„° мІҳлҰ¬ кё°мҲ  л№…лҚ°мқҙн„° кё°мҲ  (Hadoop, Spark л“ұ) лҚ°мқҙн„°лІ мқҙмҠӨ кё°мҲ  (SQL, OLAP л“ұ)
лҚ°мқҙн„° нҶөн•© лҚ°мқҙн„° нҶөн•© м—Ҷмқҙ лӢӨм–‘н•ң нҳ•нғңмқҳ лҚ°мқҙн„° м ҖмһҘ ETL н”„лЎңм„ёмҠӨлҘј нҶөн•ң лҚ°мқҙн„° нҶөн•©
лҚ°мқҙн„° л¶„м„қ лҚ°мқҙн„°лҘј н•„мҡ”н•  л•Ңк№Ңм§Җ ліҙкҙҖн•ҳкі  л¶„м„қмқ„ лӮҳмӨ‘м—җ мҲҳн–ү лҚ°мқҙн„°лҘј мӮ¬м „м—җ кө¬мЎ°нҷ”н•ҳм—¬ л¶„м„қмқ„ мҲҳн–ү
лҚ°мқҙн„° мҠӨнӮӨл§Ҳ мҠӨнӮӨл§Ҳ м—Ҷмқҙ м ҖмһҘн•ҳм—¬ мң м—°м„ұмқ„ к°–м¶Ө мӮ¬м „м—җ м •мқҳлҗң мҠӨнӮӨл§ҲлҘј л”°лҰ„
лҚ°мқҙн„° н’Ҳм§Ҳ л°Ҹ ліҙм•Ҳ лҚ°мқҙн„° н’Ҳм§Ҳ кІҖмҰқкіј ліҙм•Ҳ к°•нҷ”м—җ м–ҙл ӨмӣҖ м •нҳ•нҷ”лҗң лҚ°мқҙн„°лЎң лҚ°мқҙн„° н’Ҳм§Ҳ л°Ҹ ліҙм•Ҳ к°•нҷ”
л№„мҡ© м ҖмһҘ мҡ©лҹүм—җ л”°лҘё л№„мҡ© мҰқк°Җ кө¬мЎ°нҷ”лҗң лҚ°мқҙн„°мқҳ м ҖмһҘ л№„мҡ©
мӮ¬мҡ© мӮ¬лЎҖ л№„м •нҳ• лҚ°мқҙн„° л¶„м„қ, л№…лҚ°мқҙн„° мІҳлҰ¬, лЁёмӢ лҹ¬лӢқ лӘЁлҚё к°ңл°ң л“ұ нҶөкі„ л¶„м„қ, л№„мҰҲлӢҲмҠӨ мқён…”лҰ¬м „мҠӨ, лҰ¬нҸ¬нҢ… л“ұ

Apache Airflow

Apache Airflowм—җм„ңлҠ” python scriptлЎң DAGмқҳ кө¬мЎ°лҘј м„ӨлӘ…н•ҳкі  кө¬м„ұн•ңлӢӨ. (лҶ’мқҖ мң м—°м„ұ)
PythonмңјлЎң м •мқҳн•ң DAGлЎң pipelineмқҳ кө¬мЎ°лҘј м •мқҳн•ҳл©ҙ Apache Airflowк°Җ pipelineмқ„ м–ём ң мӢӨн–үн•  кІғмқём§Җ к°Ғк°Ғмқҳ DAGмқҳ мӢӨн–ү мЈјкё°лҘј Cronкіј к°ҷмқҖ н‘ңнҳ„мӢқмңјлЎң м •мқҳн•  мҲҳ мһҲлӢӨ.
SchedulerлҠ” м•„лһҳмҷҖ к°ҷмқҙ мһ‘лҸҷн•ңлӢӨ.

  1. м •мқҳлҗң DAG нҢҢмқј л¶„м„қ л°Ҹ к°Ғ DAG task, мқҳмЎҙм„ұ, мҳҲм•Ҫ мЈјкё° нҷ•мқё
  2. DAGмқҳ мҳҲм•Ҫ мЈјкё° кІҪкіј нҷ•мқё л°Ҹ мҳҲм•Ҫ мЈјкё°к°Җ нҳ„мһ¬ мӢңк°„ мқҙм „мқё кІҪмҡ° мӢӨн–ү мҳҲм•Ҫ
  3. мҳҲм•Ҫлҗң к°Ғ taskмқҳ мқҳмЎҙм„ұ нҷ•мқё нӣ„ мқҳмЎҙм„ұ taskк°Җ мҷ„лЈҢлҗҳм§Җ м•Ҡмқ„ мӢң execution queueм—җ м¶”к°Җ
  4. лҢҖкё° нӣ„ 1.л¶Җн„° лӢӨмӢң мӢӨн–ү

MonitoringмқҖ м•„лһҳмҷҖ к°ҷмқҙ webserverлҘј нҶөн•ҙ м§„н–үн•  мҲҳ мһҲлӢӨ.

Task мӢӨнҢЁ мӢң мһ¬мӢңлҸ„лҘј м„Өм •н•ҳм—¬ taskлҘј ліөкө¬н•  мҲҳ мһҲлӢӨ.
мһ¬мӢңлҸ„ лҳҗн•ң мӢӨнҢЁн•ҳл©ҙ task мӢӨнҢЁлҘј кё°лЎқн•ҳкі  logлҘј нҷ•мқён•ҳм—¬ debugging н•  мҲҳ мһҲлӢӨ.

Schedule кё°лҠҘ мӨ‘ DAGм—җ м •мқҳлҗң нҠ№м • мӢңм җ trigger к°ҖлҠҘн•ң кІғкіј мөңмў… мӢңм җкіј мҳҲмғҒлҗҳлҠ” лӢӨмқҢ schedule мЈјкё°лҘј мғҒм„ёнһҲ м•Ңл ӨмЈјлҠ” кІғмқҙ мЎҙмһ¬н•ңлӢӨ.
мқҙ кё°лҠҘмқ„ нҶөн•ҙ м•„лһҳмҷҖ к°ҷмқҖ кө¬м„ұмқҙ к°ҖлҠҘн•ҳлӢӨ.

  • Incremental Loading
    • л§ӨлІҲ м „мІҙ лҚ°мқҙн„° м„ёнҠёлҘј мІҳлҰ¬н•ҳм§Җ м•Ҡкі  time slotм—җ лҢҖн•ҙ м җм§„м ҒмңјлЎң мІҳлҰ¬
    • нҡЁмңЁм Ғ data pipeline кө¬м¶• к°ҖлҠҘ
  • Backfill
    • мғҲлЎң мғқм„ұн•ң DAGлҘј кіјкұ° мӢңм җ л°Ҹ кё°к°„м—җ лҢҖн•ҙ мӢӨн–ү к°ҖлҠҘ

Anatomy of an Airflow DAG

  1. DAG instance мғқм„ұмқ„ нҶөн•ң DAG м •мқҳ
  1. with DAGлҘј нҶөн•ң DAG м •мқҳ
  1. airflow.decoratorлҘј нҶөн•ң DAG м •мқҳ (TaskFlow API)

м°Ёнӣ„м—җ к°Ғ taskм—җ лҢҖн•ҙ @task decoratorлҘј мӮ¬мҡ©н•ҳм—¬ DAG к°ңл°ң нҷҳкІҪмқ„ лӢЁмҲңнҷ” н•  мҲҳ мһҲлӢӨ. (м§ҖкёҲмқҖ мқјлӢЁ л°ӣм•„л“Өмқҙкё°вҖҰ)

Scheduling in Airflow

PresetмқҖ мӢӨн–ү мЈјкё°лҘј м •мқҳн•ҳлҠ” мӮ¬м „ м •мқҳлҗң мқјл Ёмқҳ schedule к°„кІ©мқҙлӢӨ.
мқҙлҹ¬н•ң presetмқҖ DAGлҘј scheduling мӢң мЈјкё°м ҒмңјлЎң мӢӨн–үлҗҳлҠ” мқјл°ҳм Ғмқё мӢңлӮҳлҰ¬мҳӨлҘј к°„лӢЁн•ҳкІҢ м„Өм •н•ңлӢӨ.

мҳҲлҘј л“Өм–ҙ, schedule_interval="@daily"лЎң м„Өм •н•ҳл©ҙ н•ҙлӢ№ DAGлҠ” л§Өмқј мһҗм •л§ҲлӢӨ мӢӨн–үлҗңлӢӨ.
Preset мҷём—җлҸ„ м§Ғм ‘ Cron н‘ңнҳ„мӢқмқ„ мһ…л Ҙн•ҳм—¬ мӣҗн•ҳлҠ” мӢӨн–ү мЈјкё°лҘј м •нҷ•н•ҳкІҢ м§Җм •н•  мҲҳ мһҲлӢӨ.

мқҙлҹ° кІғмқ„ м°ёкі н•ҳл©ҙ мЎ°кёҲ лҚ” мүҪкІҢ к°ңл°ңмқ„ м§„н–үн•  мҲҳ мһҲлӢӨ.
л§Ҳм§Җл§үмңјлЎң datetime.timedelta()лЎң scheduleмқҳ к°„кІ©мқ„ м„Өм •н•  мҲҳ мһҲлӢӨ.
мҳҲлҘј л“Өм–ҙ, м•„лһҳмҷҖ к°ҷмқҙ н•ҳлЈЁм—җ н•ңлІҲ мӢӨн–үлҗҳлҸ„лЎқ мӢӨн–ү мЈјкё°лҘј м •мқҳн•  мҲҳ мһҲлӢӨ.

Airflowм—җм„ңлҠ” м•һм„ң л§җн•ң кІғкіј к°ҷмқҙ л§ӨлІҲ м „мІҙ лҚ°мқҙн„°лҘј мІҳлҰ¬н•ҳм§Җ м•Ҡкі  м җм§„м ҒмңјлЎң лҚ°мқҙн„°лҘј мІҳлҰ¬н•ҳлҠ”лҚ° мқҙлҘј мң„н•ҙм„ң мӢӨн–ү мӢңк°„мқ„ м°ёмЎ°н•ҙм•јн•ңлӢӨ.
мқҙ мӢӨн–ү мӢңк°„ (execution_date)мқҖ мӢӨм ң мӢӨн–үлҗҳлҠ” мӢңк°„мқҙ м•„лӢҢ лҚ°мқҙн„°к°Җ мӢӨм ң мІҳлҰ¬лҗҳм—ҲлӢӨкі  к°Җм •н•ң мӢңм җмқҙлӢӨ.
м•„лһҳмқҳ мҳҲмӢңлҠ” "10 * * * *"мқҳ мЈјкё°лЎң мӢӨн–үлҗҳлҠ”лҚ°, мӢӨм ң мӢӨн–ү мӢңк°„мқё Start DateмҷҖ End DateлҠ” мӮ¬м§„ кё°мӨҖмқҳ нҳ„мһ¬ мӢңк°„мһ„мқ„ м•Ң мҲҳ мһҲлӢӨ.
н•ҳм§Җл§Ң Logical Date (execution_date, ts)лҠ” Cronмқҳ schedule к°„кІ©мқё 1мӢңк°„ м „мңјлЎң лӘ…мӢңлҗҳм–ҙмһҲмқҢмқ„ м•Ң мҲҳ мһҲлӢӨ.

мқҙкІғмқҖ tsк°Җ schedule к°„кІ©мқҳ мӢңмһ‘ мӢңм җмқ„ мқҳлҜён•ҳкё° л•Ңл¬ёмқҙлӢӨ.
